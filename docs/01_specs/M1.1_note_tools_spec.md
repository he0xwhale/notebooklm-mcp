# Spec M1.1: NotebookLM Note Listing and Retrieval

## 1. Background

Users of NotebookLM MCP need a way to inspect the titles of all saved notes inside their active notebook and extract their full text content dynamically. This allows external tools (like Claude) to read existing annotations and utilize them for RAG or other reasoning tasks.

## 2. Requirements

- **`note_list` / `list_notes`**:
  - Automatically navigate to the Studio tab of the current notebook.
  - Scroll down the Studio panel to ensure lazy-loaded note cards are rendered.
  - Scrape note cards matching the tag `artifact-library-note` and class `[class*="artifact-item-button"]`.
  - Extract the title (`.artifact-title`), timestamp (`.artifact-details`), and unique ID (`data-note-id`).
  - Return a list of all identified notes (success, titles, IDs, details).
- **`note_get` / `get_note`**:
  - Search for a specific note element using `findNoteElement` with the note title or note ID.
  - Support the new verified DOM selectors (`artifact-library-note`, `.artifact-title`).
  - Click on the note card to open its editor.
  - Extract text content from the ProseMirror editor or alternative fallback selectors.
  - Support close-on-finish behavior by pressing the Escape key to return the UI to a clean state.
  - Return the note title and text content.
